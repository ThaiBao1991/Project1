    Day 22 — Case Merge Table (Ghép bảng thực chiến)

    1. Concept (30%)

    Trong thực tế (ví dụ inspection lens, sản lượng, lỗi…), dữ liệu hiếm khi nằm trong 1 bảng.
    Bạn thường có:
    • Bảng sản lượng (Production)
    • Bảng lỗi (Defect)
    • Bảng thông tin line / model
    👉 Merge Table = join dữ liệu từ nhiều bảng lại theo key chung
    Giống như:
    • Bạn có 1 bảng "số lượng lens sản xuất"
    • Và 1 bảng "số lượng lỗi"
    • → Merge lại để biết tỷ lệ lỗi theo line/ngày/model
    💡 Trọng tâm hôm nay:
    • Table.NestedJoin
    • Expand bảng sau merge
    • Kết hợp logic nhiều nguồn

    2. Code Breakdown (40%)

    🎯 Scenario:

    Bạn có 2 bảng:
    Table Production
    DateLineOutput
    2024-01-01A1000
    2024-01-01B800
    Table Defect
    DateLineDefectQty
    2024-01-01A50
    2024-01-01B40
    👉 Goal: Merge → tính Defect Rate

    ✅ M Code hoàn chỉnh

    Power Query
    let
    // Source bảng production
    SourceProd = Production,

    // Source bảng defect
    SourceDef = Defect,

    // Merge 2 bảng theo Date + Line
    MergeTable = Table.NestedJoin(
    SourceProd,
    {"Date", "Line"},
    SourceDef,
    {"Date", "Line"},
    "DefectData",
    JoinKind.LeftOuter
    ),

    // Expand dữ liệu từ bảng Defect
    ExpandDefect = Table.ExpandTableColumn(
    MergeTable,
    "DefectData",
    {"DefectQty"}
    ),

    // Thêm cột Defect Rate
    AddRate = Table.AddColumn(
    ExpandDefect,
    "DefectRate",
    each [DefectQty] / [Output],
    type number
    )
    in
    AddRate
    ``

    🔍 Giải thích từng phần

    1. Table.NestedJoin

    Power Query
    Table.NestedJoin(SourceProd, {"Date", "Line"}, SourceDef, {"Date", "Line"}, "DefectData", JoinKind.LeftOuter)
    • Join 2 bảng bằng 2 key: Date + Line
    • Kết quả:
      • Không flatten ngay
      • Tạo ra column dạng table (nested table)
    👉 Mỗi dòng Production sẽ chứa:
    DefectData = [ bảng nhỏ chứa DefectQty ]

    2. Table.ExpandTableColumn

    Power Query
    Table.ExpandTableColumn(MergeTable, "DefectData", {"DefectQty"})
    • Bung nested table ra thành column thật
    • Đây là bước bắt buộc sau merge nếu cần lấy dữ liệu

    3. Table.AddColumn

    Power Query
    each [DefectQty] / [Output]
    • Dùng dữ liệu từ 2 bảng đã merge
    • Tính KPI: tỷ lệ lỗi

    ⚠️ Lưu ý thực tế (rất quan trọng)

    1. Join type
      • LeftOuter: giữ toàn bộ Production
      • Inner: chỉ giữ dòng match
      • Chọn sai → mất dữ liệu
    2. Null handling
      • Nếu Defect không có → sẽ = null
      • Nên xử lý:
    Power Query
    each if [DefectQty] = null then 0 else [DefectQty]

    3. Practice (30%)

    🟢 Bài 1 (Cơ bản)

    Merge 2 bảng:
    • Orders (OrderID, CustomerID)
    • Customers (CustomerID, Name)
    👉 Yêu cầu:
    • Merge
    • Expand cột Name

    🟡 Bài 2 (Trung cấp)

    Bạn có:
    • Production (Date, Line, Output)
    • Downtime (Date, Line, Minutes)
    👉 Yêu cầu:
    • Merge theo Date + Line
    • Tính:
    Downtime per unit = Minutes / Output

    🔴 Bài 3 (Thực chiến)

    Bạn có 3 bảng:
    • Production
    • Defect
    • Target (Line, TargetRate)
    👉 Yêu cầu:
    1. Merge Production + Defect
    2. Merge tiếp với Target
    3. Tính:
    Performance = DefectRate / TargetRate

    ✅ Kết luận

    Hôm nay bạn đã nắm kỹ năng cực quan trọng:
    👉 Merge table + xử lý nested data + tính toán sau khi join
    Đây là nền tảng cho:
    • Báo cáo KPI
    • Ghép data nhiều nguồn
    • Dashboard thực tế
